1. Generalidades

1.1 Introducción

El diseño de un edificio con análisis sísmico es fundamental para garantizar la seguridad y estabilidad de las edificaciones frente a los efectos de un terremoto. Este proyecto tiene como objetivo automatizar este proceso mediante herramientas computacionales que permitan realizar los cálculos necesarios de manera rápida y precisa.

1.2 Antecedentes

El diseño con un análisis sísmico es una disciplina esencial para una estructura de gran altura, especialmente en zonas de alta actividad sísmica. Existen diferentes métodos y normativas para realizar estos análisis, siendo uno de los más utilizados el análisis de estructuras con el método de espectro de respuesta. Sin embargo, la automatización de estos cálculos sigue siendo un reto técnico que este proyecto busca resolver.

1.3 Planteamiento del Problema

1.3.1 Identificación del problema

En la actualidad, los cálculos sísmicos para el diseño de edificaciones de hormigón armado suelen realizarse de manera manual o utilizando programas que requieren de entradas complejas y cálculos extensos. Este proceso puede ser tedioso y propenso a errores humanos. Además, la falta de automatización en la fase de diseño impide realizar simulaciones rápidas y precisas para diferentes escenarios sísmicos. El reto del proyecto es automatizar este proceso para facilitar el diseño estructural en regiones sísmicamente activas.

1.5 Objetivo y acciones

1.5.1 Objetivo General

Desarrollar y automatizar el análisis sísmico de edificaciones en estructuras de hormigón armado, con el fin de optimizar el proceso de diseño y mejorar la seguridad de las edificaciones frente a terremotos.

1.5.2 Objetivos Específicos

  • Automatizar el cálculo de las fuerzas sísmicas aplicadas a las estructuras.
  • Desarrollar una herramienta computacional basada en un modelo estructural que permita calcular la respuesta sísmica.
  • Implementar un sistema que integre el análisis sísmico con el diseño estructural en tiempo real.

1.6 Justificaciones

1.6.1 Justificación Técnica

La justificación técnica de este proyecto se basa en la optimización de los procesos de diseño estructural y el análisis sísmico. La automatización de estos cálculos no solo mejora la eficiencia del diseño, sino que también aumenta la precisión al minimizar errores humanos. Además, al integrar estos cálculos con herramientas modernas, se agiliza el proceso y se permite evaluar rápidamente diversas configuraciones estructurales bajo distintos escenarios sísmicos.

1.6.2 Justificación Económica

El desarrollo de esta herramienta de automatización permitirá reducir los tiempos de cálculo, lo que se traduce en menores costos para los proyectos de construcción. La reducción en el tiempo de diseño también implica una disminución en los costos operativos y el tiempo de entrega del proyecto. Esto resulta en una ventaja competitiva para las empresas de construcción que adopten esta tecnología.

1.7 Alcance y Limitaciones

1.7.1 Alcance Geográfico

El alcance geográfico de este proyecto está centrado en Bolivia, específicamente en regiones de alta actividad sísmica como Santa Cruz y Cochabamba, donde los terremotos representan una amenaza constante. A pesar de que el enfoque es local, la metodología desarrollada podría ser adaptada a otros países de América Latina con características sísmicas similares.

1.7.2 Alcance Temporal

Este proyecto tiene un tiempo estimado de desarrollo de 6 meses, durante los cuales se llevará a cabo la investigación, desarrollo y pruebas de la herramienta. La implementación y pruebas iniciales se realizarán en una estructura de edificio modelo para validar la eficacia de la herramienta automatizada.

1.7.3 Limitaciones

Entre las principales limitaciones del proyecto se incluyen la dependencia de la calidad de los datos ingresados, la necesidad de un software compatible con el que se desarrolle la herramienta de análisis y las limitaciones tecnológicas en el procesamiento de grandes volúmenes de datos sísmicos en tiempo real.

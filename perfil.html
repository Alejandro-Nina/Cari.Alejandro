1. Generalidades

1.1 Introducción

El diseño de un edificio con análisis sísmico es esencial para garantizar la seguridad y estabilidad de las edificaciones frente a los efectos de un terremoto. Este proyecto busca automatizar este proceso utilizando herramientas computacionales que permitan realizar cálculos rápidos y precisos.

1.2 Antecedentes

En zonas de alta actividad sísmica, el diseño de estructuras con análisis sísmico es crucial para evitar colapsos. Sin embargo, la falta de automatización del proceso sigue siendo un reto. Este proyecto tiene como objetivo automatizar los cálculos de diseño sísmico, acelerando el proceso y reduciendo los errores humanos.

1.3 Planteamiento del Problema

1.3.1 Identificación del problema

Actualmente, los cálculos sísmicos se realizan manualmente o mediante programas con procesos complejos. Este proyecto busca automatizar el análisis sísmico, minimizando los errores humanos y acelerando el proceso de diseño. El reto es integrar esta automatización en un sistema accesible y eficiente.

1.5 Objetivo y acciones

1.5.1 Objetivo General

Desarrollar una herramienta computacional que automatice el análisis sísmico de edificaciones de hormigón armado, mejorando la eficiencia y precisión del diseño estructural.

1.5.2 Objetivos Específicos

  • Automatizar el cálculo de las fuerzas sísmicas aplicadas a las estructuras.
  • Crear una herramienta que permita realizar simulaciones rápidas de la respuesta sísmica de diferentes estructuras.
  • Integrar el análisis sísmico con el diseño estructural en un sistema automatizado.

1.6 Justificaciones

1.6.1 Justificación Técnica

Este proyecto permite la automatización del proceso de diseño estructural y análisis sísmico, reduciendo el tiempo de cálculo y aumentando la precisión al evitar errores humanos. El sistema facilitará el análisis de diversas configuraciones estructurales bajo diferentes condiciones sísmicas.

1.6.2 Justificación Económica

La automatización reduce el tiempo de diseño, lo que implica menores costos operativos y una entrega más rápida de los proyectos. Las empresas de construcción que adopten esta tecnología tendrán una ventaja competitiva en el mercado.

1.7 Alcance y Limitaciones

1.7.1 Alcance Geográfico

El proyecto está centrado en Bolivia, particularmente en regiones sísmicamente activas como Santa Cruz y Cochabamba. Aunque el enfoque es local, la metodología podría adaptarse a otros países de América Latina con características sísmicas similares.

1.7.2 Alcance Temporal

El proyecto tiene un tiempo estimado de desarrollo de 6 meses, durante los cuales se realizarán pruebas y validaciones de la herramienta en una estructura de ejemplo.

1.7.3 Limitaciones

Las limitaciones incluyen la necesidad de datos precisos para el análisis sísmico y la capacidad del software para manejar grandes volúmenes de datos en tiempo real.
